RoATP Assessor Journey

Public Beta - Staff App v12 - Assessor IN REVIEW
Created on: 6th Apr 2020
Last updated: 28nd Apr 2020
User research:

Background information (README)

The prototype was last updated in Apr 2020.

Assessor

  • Ability to assign yourself as an assessor of an application
  • Ability for two assessors to assign themselves to an application in order to carry out a blind assessment
  • An application will appear in the 'New' tab IF there is at least one assessor space available AND that you are not an assessor assigned to the application already
  • An application will appear in the 'In progress' tab IF you are the only assessor on the application OR both assessor spots are filled
  • When an assessment is in the 'In progress' tab, it will remain there until both assessors complete their blind assessment. However, when an assessor has completed their blind assessment but the other assessor has not, the link to the application will take them back to the confirmation page they saw when they submitted their assessment checks.
  • An application will appear in the 'Moderation' tab IF both assessors have completed their blind assessments
  • The task list shown when clicking 'Moderate application' from the 'Moderation' tab will:
    • Display all sections that were available to the assessor, showing a pass / fail against sections which have a consensus reached by the two assessors. No watermark tag will be against sections which have disparity between the assessors.
    • Moderating a section will display the original question, the applicants answer, the guidance shown to the applicant in the apply journey, the outcome and comments of both blind assessments, and the ability to record an outcome to the moderation
  • When 'failing' a question in 'moderate, the assessor must give an internal comment and an external comment (for use for clarification feedback or for provider feedback post-oversight).
  • When a clarification has been sent, it shows as 'with applicant' and the assessor can change the overall outcome of the application at any point - but they should only do so when a clarification has not been received. For MVS, we aren't building a overdue flag in the service to know when the application has been with the applicant for longer than 5 working days.
  • When a clarification comes back from the provider, the assessor can see the clarified questions, with a view of the previous answer, the new answer and the internal and external comment that was logged. They can also change the outcome on any other questions, and send an overall outcome for the application when they have reviewed all clarified questions, where they can pass / fail / ask for clarification (again).

  • Staff App v11 - Gateway & Assessor NON-MVS
    Created on: 8th Jan 2020
    Last updated: 14th Jan 2020
    User research: 22nd to 23rd Jan 2020
    Background information (README)

    The prototype was last updated in Jan 2020, at which point it was deemed that the functionality that we were designing in this version of the prototype was deemed to be 'non-MVS'.

    Due to both technological and time constraints placed upon the project at this point, many of the features required by the different assessment teams were de-scoped in order to deliver to deadlines.

    The features that this prototype demonstrates that have since been removed in the newer MVS versions (and therefore will not be delivered as the first release of the Private Beta phase) are:

    Gateway

    Accessed by signing in with no credentials

    • Ability to view a read-only version of an application
    • Ability for a Gateway user to assign an application to themselves
    • Ability to select a question level yes/no to identify (for reporting and the applicant in case of requested clarifications) which questions require attention
    • Ability to assign an application already in progress to the user who is logged in
    • Ability to search applications by UKPRN or organisation name

    Assessor

    Accessed by signing in with assessor@esfa.gov.uk email, no password

    • Ability to view a read-only version of an application
    • Ability to assign yourself as an assessor of an application
    • Ability for two assessors to assign themselves to an application in order to carry out a blind assessment
    • Ability for an assessor to replace another specific assessor on an application in progress
    • An application will appear in the 'New' tab IF there is at least one assessor space available AND that you are not an assessor assigned to the application already
    • An application will appear in the 'In progress' tab IF you are the only assessor on the application OR both assessor spots are filled
    • An application will appear in the 'Moderation' tab IF both assessors have completed their blind assessments
    • The task list shown when clicking 'Moderate application' from the 'Moderation' tab will:
      • Display all sections that the assessor deals with, but the links will only be active on the sections on where at least one of the assessor had failed a question
      • Any links where the blind assessments both agree on a pass will not be available to moderate
      • Moderating a section will display the original question, the applicants answer, the guidance shown to the applicant in the apply journey, the outcome and comments of both blind assessments, and the ability to record an outcome to the moderation
    • An application will appear in the 'Clarifications' tab IF the outcome of the moderation process requires further information from the applicant
    • An application will appear in the 'Outcome' tab IF the moderation process results in a 'Fail' or a 'Pass'
    • Ability to search applications by UKPRN or organisation name
